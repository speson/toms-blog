# 포스트 발행

`.claude/news/` 파일에서 체크된 뉴스를 기반으로 MDX 블로그 포스트를 작성합니다.

## 사용 방법

```
/publish              # 가장 최근 뉴스 파일에서 발행
/publish 20260123     # 특정 날짜 파일에서 발행
/publish --draft      # 초안으로 저장 (draft: true)
```

인자: `${ARGUMENTS}`

---

## Step 1: 뉴스 파일 찾기

### 인자가 없는 경우

가장 최근 뉴스 파일을 찾습니다:

```bash
ls -t .claude/news/*.md | head -1
```

### 날짜가 지정된 경우

해당 날짜의 파일을 찾습니다:
- 입력: `20260123` → 파일: `.claude/news/20260123_news.md`

### 파일이 없는 경우

```
⚠️ 뉴스 파일이 없습니다.
먼저 /fetch-news를 실행하세요.
```

---

## Step 2: 체크된 항목 파싱

뉴스 파일에서 `[x]`로 체크된 항목만 추출합니다.

### 파싱 규칙

```markdown
- [x] **제목**        ← 체크됨, 발행 대상
  - 날짜: ...
  - URL: ...
  - 요약: ...

- [ ] **제목**        ← 체크 안됨, 제외
  - 날짜: ...
```

### 체크된 항목이 없는 경우

```
⚠️ 체크된 뉴스가 없습니다.
뉴스 파일에서 발행할 항목을 [x]로 체크하세요.

파일: .claude/news/YYYYMMDD_news.md
```

---

## Step 3: 체크된 항목 확인

파싱된 항목을 사용자에게 표시합니다:

```
📰 발행할 뉴스 (N개)

1. 제목1 (geeknews)
2. 제목2 (github)
...

계속 진행할까요?
```

---

## Step 4: 포스트 유형 결정

### 단일 뉴스 (1개 체크)
- 해당 뉴스의 번역/요약 포스트 작성
- 제목: 원문 제목 번역

### 복수 뉴스 (2개 이상 체크)
- 뉴스 모음 형식의 포스트 작성
- 제목: "AI 뉴스 모음 - YYYY년 M월 D일" 형식

---

## Step 5: 포스트 내용 작성

### 단일 뉴스 포스트 템플릿

```mdx
---
title: "번역된 제목"
description: "1-2문장 요약"
date: "YYYY-MM-DD"
tags: ["AI", "관련태그"]
source: "geeknews" 또는 "github"
sourceUrl: "원문 URL"
draft: false
---

## 요약

핵심 내용 3-5문장 요약

## 주요 내용

### 섹션 1
내용...

### 섹션 2
내용...

## 원문 링크

- [원문 제목](원문URL)
```

### 복수 뉴스 포스트 템플릿

```mdx
---
title: "AI 뉴스 모음 - YYYY년 M월 D일"
description: "오늘의 AI/개발 뉴스 N개를 정리했습니다."
date: "YYYY-MM-DD"
tags: ["AI", "뉴스", "정리"]
source: "curated"
draft: false
---

오늘의 AI/개발 관련 뉴스를 정리했습니다.

## 1. 첫 번째 뉴스 제목

> 원문 요약 또는 핵심 내용

### 주요 포인트
- 포인트 1
- 포인트 2

[원문 보기](URL)

---

## 2. 두 번째 뉴스 제목

> 원문 요약

### 주요 포인트
- 포인트 1
- 포인트 2

[원문 보기](URL)

---

(반복...)
```

---

## Step 6: 파일명 생성

파일명 규칙:
- 형식: `content/posts/{slug}.mdx`
- slug: 날짜-키워드 조합 (영문, kebab-case)

예시:
- 단일: `2026-01-23-openai-gpt-5-release.mdx`
- 복수: `2026-01-23-ai-news-roundup.mdx`

---

## Step 7: 파일 저장

MDX 파일을 `content/posts/` 디렉토리에 저장합니다.

---

## Step 8: 빌드 검증

```bash
pnpm build
```

빌드가 성공하면 완료 메시지:

```
✅ 포스트 발행 완료!

📄 파일: content/posts/{slug}.mdx
🔗 URL: /posts/{slug}

다음 단계:
- pnpm dev 로 미리보기
- /commit 으로 커밋
- /commit --push 로 배포
```

---

## Step 9: 뉴스 파일 정리 (선택)

발행 완료 후 뉴스 파일의 체크된 항목을 `[x]` → `[✓]`로 변경하여 발행 완료 표시:

```markdown
- [✓] **제목** (발행됨)
```

## 옵션

| 옵션 | 설명 |
|------|------|
| `--draft` | 초안으로 저장 (draft: true), 사이트에 노출되지 않음 |

## 작성 가이드라인

### 번역 원칙
- 자연스러운 한국어로 의역
- 기술 용어는 영문 병기 (예: "대규모 언어 모델(LLM)")
- 고유명사는 원문 유지 (예: OpenAI, Claude)

### 태그 규칙
- 최대 5개
- 소문자, 한글 또는 영문
- 일관된 태그 사용: AI, LLM, OpenAI, 오픈소스, 개발도구 등

### 내용 구성
- 핵심 먼저 (inverted pyramid)
- 불릿 포인트 적극 활용
- 코드 블록은 필요한 경우만
