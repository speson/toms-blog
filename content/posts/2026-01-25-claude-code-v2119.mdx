안녕하세요, Tom입니다.

Claude Code가 **v2.1.19**로 업데이트되었습니다! 이번 버전은 태스크 관리 시스템 개선과 VSCode 사용자를 위한 기능이 주를 이룹니다.

## 주요 변경사항

### 새로운 기능

| 기능                       | 설명                                                   |
| -------------------------- | ------------------------------------------------------ |
| `CLAUDE_CODE_ENABLE_TASKS` | 환경 변수로 태스크 관리 시스템 비활성화 가능           |
| 인덱스 인수 단축키         | 커스텀 명령어에서 `$0`, `$1` 등으로 개별 인수 접근     |
| 세션 포크/되감기           | VSCode 사용자 전체에게 세션 포크 및 되감기 기능 활성화 |

💡 **세션 포크가 뭔가요?** 현재 세션을 복제해서 새로운 브랜치에서 작업할 수 있는 기능입니다. 실험적인 작업을 할 때 유용합니다.

### 버그 수정 (제가 주목한 것들)

**1. AVX 미지원 프로세서 크래시 수정**

v2.1.17에서 시작된 이슈였는데, 드디어 수정되었습니다. 오래된 CPU를 쓰시는 분들은 이제 안정적으로 쓸 수 있어요.

**2. 터미널 종료 시 프로세스 남는 문제 수정**

🎯 **이거 진짜 짜증났어요.** 터미널을 닫아도 Claude Code 프로세스가 계속 살아있어서 메모리를 잡아먹는 문제가 있었는데, 이제 깔끔하게 종료됩니다.

**3. 프롬프트 스태시(Ctrl+S) 사용 시 텍스트 손실 문제 수정**

긴 프롬프트를 작성하다가 Ctrl+S로 스태시하고 붙여넣으면 텍스트가 날아갔는데, 이제 안전합니다.

### 변경사항

- 추가 권한이나 훅이 없는 스킬은 승인 없이 허용
- 인덱스 인수 문법 변경: `$ARGUMENTS.0` → `$ARGUMENTS[0]` (브라켓 문법)

⚠️ **주의:** 커스텀 명령어에서 인덱스 인수 문법이 바뀌었습니다. 기존에 `$ARGUMENTS.0` 형식으로 쓰셨다면 `$ARGUMENTS[0]`로 수정하세요.

## 업데이트 방법

```bash
# npm
npm install -g @anthropic-ai/claude-code@latest

# 또는
claude-code --update
```

## 제 생각

이번 업데이트는 **안정성 개선**에 초점이 맞춰져 있습니다. 새로운 기능보다는 기존 기능의 버그를 잡는 데 주력했네요.

💰 **월 $20 내고 쓰시는 분들:** 이런 꾸준한 업데이트가 Claude Code의 가치를 높입니다. 버그가 빠르게 수정되고, 사용자 피드백이 반영되는 게 보이죠.

특히 터미널 종료 시 프로세스 남는 문제 수정은 제가 직접 겪었던 문제라서 반가웠습니다. 이제 Claude Code를 더 안심하고 쓸 수 있을 것 같아요.
