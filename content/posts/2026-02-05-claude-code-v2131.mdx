안녕하세요, Tom입니다.

**Claude Code v2.1.31**이 나왔어요! 작지만 실용적인 개선들이 가득합니다. 특히 **세션 재개 경험**이 훨씬 좋아졌어요.

## 세션 재개 힌트 표시 💬

이제 Claude Code를 종료할 때 **나중에 대화를 이어가는 방법**을 알려줍니다.

```
👋 Goodbye! To continue this conversation later, run:
   claude --resume ses_abc123xyz
```

💡 **왜 이게 유용한가:**

이전에는 세션 ID를 기억해야 했는데, 이제는 종료 메시지에 바로 보여줘서 복사-붙여넣기만 하면 됩니다. 작지만 큰 편의성 개선이에요.

🎯 **실제로 써보니:**

긴 작업을 하다가 점심 먹으러 가야 할 때, 종료 메시지만 터미널에 남겨두면 돌아와서 바로 이어서 할 수 있어요. 예전에는 `claude --list-sessions`로 ID를 찾아야 했거든요.

## 일본어 입력 지원 🇯🇵

체크박스 선택에서 **전각(zenkaku) 스페이스 입력**을 지원해요.

일본어 IME를 쓰는 분들에게는 반가운 소식이죠. 이전에는 전각 스페이스가 입력되지 않아서 체크박스 선택이 안 되는 문제가 있었습니다.

## 중요한 버그 수정들 🐛

### PDF 락업 문제 해결

**"PDF too large" 에러가 발생하면 세션이 영구적으로 멈춰버리던 문제**가 해결됐어요.

⚠️ **이전 문제:** 큰 PDF를 읽으려고 하면 에러가 나고, 그 세션은 다시는 못 쓰게 됐습니다. 새 세션을 시작해야 했어요.

✅ **해결:** 이제 에러가 나도 세션은 계속 사용할 수 있습니다.

### Sandbox 모드 버그 수정

Sandbox 모드가 켜져 있을 때 bash 명령이 **"Read-only file system" 에러**를 잘못 리포트하던 문제가 수정됐어요.

### Plan 모드 크래시 해결

`~/.claude.json`에 프로젝트 설정의 기본 필드가 없을 때 plan 모드에 진입하면 크래시가 나던 문제가 해결됐습니다.

### temperatureOverride 버그

스트리밍 API에서 `temperatureOverride` 설정이 **완전히 무시되던 버그**가 수정됐어요.

⚠️ **이전:** 설정에서 temperature를 0.7로 바꿔도 계속 기본값(1)이 사용됐습니다.

✅ **해결:** 이제 제대로 적용됩니다.

## 시스템 프롬프트 개선 📝

Claude가 bash 명령 대신 **전용 도구를 더 잘 사용하도록** 프롬프트가 개선됐어요.

| Bash 명령 | 전용 도구 |
| --------- | --------- |
| `cat`     | Read      |
| `sed`     | Edit      |
| `grep`    | Grep      |
| `find`    | Glob      |

💰 **토큰 절약:** 전용 도구를 쓰면 bash 명령보다 토큰 사용량이 적고, 결과도 더 정확합니다.

## 기타 개선 사항

### 에러 메시지 개선

PDF와 요청 크기 제한을 명확히 표시합니다:

- PDF: 최대 100페이지
- 요청 크기: 최대 20MB

### 터미널 레이아웃 개선

스트리밍 중에 스피너가 나타나고 사라질 때 **화면이 덜 떨리게** 개선됐어요.

### 써드파티 프로바이더 개선

Bedrock, Vertex, Foundry 사용자에게는 더 이상 **Anthropic API 가격이 표시되지 않습니다**.

## 총평

v2.1.31은 **"사용성 개선의 정석"**입니다. 세션 재개 힌트 하나만으로도 일상적인 사용이 훨씬 편해졌어요.

제가 느낀 점:

- 세션 재개 힌트 → 워크플로우 개선
- PDF 락업 수정 → 안정성 향상
- temperatureOverride 버그 수정 → 설정이 제대로 작동
- 시스템 프롬프트 개선 → 더 효율적인 도구 사용

작은 업데이트처럼 보이지만, 실제로 써보면 체감되는 개선들이 많습니다. 업데이트 추천드려요!

```bash
# 업데이트 방법
claude --update
```
