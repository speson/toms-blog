안녕하세요, Tom입니다.

**Oh My OpenCode** v3.2.2가 출시되었습니다! 이번 업데이트는 **GPT-5.2 프롬프트 최적화**와 **모델 기반 라우팅** 기능이 핵심입니다.

## 주요 변경사항

### 1. GPT-5.2 Prompt Optimization 🚀

Atlas, Sisyphus-Junior, Oracle 에이전트들이 이제 GPT-5.2에 최적화된 프롬프트를 사용합니다.

💡 **XML 구조 + 상세한 verbos 프롬프트:**

- 더 정확한 응답
- 일관된 출력 형식
- 복잡한 작업의 품질 향상

🎯 **실제 차이:**

- 이전: "코드를 개선해줘"
- 이후: "<task>코드 개선</task><context>React 컴포넌트</context><requirements>성능 최적화, 가독성 향상</requirements>"

### 2. Model-based Prompt Routing

에이전트별로 최적의 모델과 프롬프트 조합을 자동으로 선택합니다.

| 에이전트        | 모델    | 프롬프트 스타일     |
| --------------- | ------- | ------------------- |
| Atlas           | GPT-5.2 | 오케스트레이션 중심 |
| Sisyphus-Junior | GPT-5.2 | 작업 실행 중심      |
| Oracle          | GPT-5.2 | 분석/조언 중심      |

⚠️ **이게 중요합니다:** 작업 유형에 따라 최적의 모델-프롬프트 조합을 자동으로 선택해서, 토큰 효율성과 응답 품질이 모두 향상됩니다.

## 업데이트 방법

```bash
# npm
npm update -g oh-my-opencode

# Homebrew
brew upgrade oh-my-opencode
```

## 제 생각

v3.2.2는 "**프롬프트 엔지니어링의 진화**"를 보여줍니다. 단순히 모델을 바꾸는 게 아니라, **프롬프트 구조 자체를 최적화**한 거죠.

💰 **비용 효율성:** GPT-5.2는 성능이 좋지만 비용도 높습니다. Oh My OpenCode는 필요한 때만 GPT-5.2를 사용하고, 다른 작업은 더 저렴한 모델로 라우팅해서 **비용을 최적화**합니다.

**주의할 점:** GPT-5.2 프롬프트는 더 길어서, 짧은 작업에서는 오히려 토큰 비용이 늘어날 수 있습니다. 하지만 복잡한 작업에서는 품질 향상으로 인해 **전체 작업 시간이 단축**되어 비용이 절감됩니다.

## 마무리

Oh My OpenCode v3.2.2는 **멀티 에이전트 오케스트레이션의 새로운 기준**을 제시합니다. GPT-5.2의 강력한 성능을 효율적으로 활용하는 방법을 보여주죠.
