안녕하세요, Tom입니다.

Claude Code가 **v2.1.16**으로 업데이트되었습니다! 이번 버전은 생산성 향상에 초점이 맞춰져 있어요. 특히 **태스크 관리 시스템**과 **VSCode 플러그인 관리** 기능이 눈에 띄더라고요.

## 새로운 기능

### 1. 태스크 관리 시스템 🎯

이게 제일 기대됐어요. 이제 Claude Code가 **의존성 추적(dependency tracking)**을 지원합니다.

예를 들어:

- "A 기능을 먼저 구현하고"
- "A가 완료되면 B 기능을 구현해"

이런 식으로 태스크 간 관계를 명시적으로 정의할 수 있어요. 복잡한 프로젝트에서 작업 흐름 관리가 훨씬 쉬워지겠죠?

💡 **실제 사용 예시:**

```
[태스크 1] API 엔드포인트 구현
  ↓ (의존)
[태스크 2] 프론트엔드 연동
  ↓ (의존)
[태스크 3] 테스트 작성
```

### 2. VSCode 네이티브 플러그인 관리

VSCode 사용자들은 이제 Claude Code 내에서 **직접 플러그인을 설치하고 관리**할 수 있습니다. 별도로 VSCode 설정을 들어가지 않아도 되서 편하더라고요.

### 3. 원격 세션 복원 (OAuth 사용자)

Sessions 다이얼로그에서 **원격 Claude 세션을 탐색하고 재개**할 수 있게 됐어요. OAuth로 로그인한 사용자만 해당됩니다.

⚠️ **참고:** 아직 API 키 방식으로 쓰시는 분들은 이 기능을 쓸 수 없어요. OAuth 전환을 고려해보세요.

## 버그 수정

| 문제                                           | 상태    |
| ---------------------------------------------- | ------- |
| 서브에이전트 과다 사용 세션 재개 시 OOM 크래시 | ✅ 수정 |
| `/compact` 후 "context remaining" 경고 미숨김  | ✅ 수정 |
| 세션 타이틀이 사용자 언어 설정 무시            | ✅ 수정 |
| Windows: 사이드바 컨테이너 시작 시 미표시      | ✅ 수정 |

🎯 **OOM 크래시 수정이 중요해요.** 복잡한 작업을 하다가 세션을 재개하면 메모리 부족으로 터지는 경우가 있었는데, 이제 안정적입니다.
