안녕하세요, Tom입니다.

**Oh My OpenCode** v3.1.7이 릴리스되었습니다! MCP 서버를 위한 **OAuth 2.1 완전 지원**과 **LSP 클라이언트 마이그레이션**이 주요 변경사항입니다.

## OAuth 2.1 완전 지원

MCP(Model Context Protocol) 서버를 위한 **OAuth 2.1 인증**이 완전히 구현되었습니다.

### 지원 RFC 표준

| RFC          | 설명                                                   |
| ------------ | ------------------------------------------------------ |
| **RFC 7591** | Dynamic Client Registration (동적 클라이언트 등록)     |
| **RFC 9728** | Protected Resource Metadata (보호된 리소스 메타데이터) |
| **RFC 8414** | Authorization Server Discovery (인증 서버 디스커버리)  |
| **RFC 8707** | Resource Indicators (리소스 표시자)                    |

💡 **이게 왜 중요한가:** MCP 서버 연결 시 인증 설정이 더 명확하고 표준화되었습니다.

### 주요 기능

- **보안 토큰 스토리지**: 안전한 토큰 저장소
- **Step-up Authorization**: 단계별 권한 상승
- **CLI 명령어**:
  ```bash
  opencode mcp oauth login   # MCP OAuth 로그인
  opencode mcp oauth logout  # OAuth 로그아웃
  opencode mcp oauth status  # 현재 OAuth 상태 확인
  ```

🎯 **실제 사용 예시:**

```bash
# MCP 서버에 OAuth로 로그인
opencode mcp oauth login --server my-mcp-server

# 토큰 확인
opencode mcp oauth status
```

## LSP 클라이언트 마이그레이션

기존 커스텀 JSON-RPC 구현을 **vscode-jsonrpc 라이브러리**로 마이그레이션했습니다.

### 개선사항

- **안정성**: Microsoft의 검증된 라이브러리 사용
- **호환성**: VSCode 생태계와의 더 나은 통합
- **유지보수**: 커스텀 코드 감소로 버그 가능성 감소

⚠️ **주의:** 마이그레이션으로 인해 일부 edge case에서 동작이 달라질 수 있습니다. 문제 발생 시 이슈를 제보해주세요.

## 백그라운드 에이전트 안정성 개선

### 좀비 프로세스 버그 수정

이전 버전에서 백그라운드 에이전트가 종료되지 않고 좀비 프로세스로 남는 문제가 있었습니다. v3.1.7에서 이 문제가 완전히 수정되었습니다.

🎯 **이게 중요합니다:** 장시간 작업 시 메모리 누수가 방지됩니다.

## 제가 실제로 써본 결과

**1. OAuth 설정이 훨씬 간단해졌어요**

이전에는 MCP 서버 인증 설정이 복잡했는데, 이제는 `opencode mcp oauth login` 한 번이면 됩니다.

**2. LSP 통신이 더 안정적입니다**

vscode-jsonrpc로 바뀌면서, 가끔 발생하던 LSP 연결 끊김 문제가 사라졌어요.

**3. 백그라운드 작업이 믿을 만해요**

좀비 프로세스 문제가 해결되어, 백그라운드에서 장시간 작업을 맡길 수 있게 됐습니다.
