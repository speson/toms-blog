안녕하세요, Tom입니다.

**Oh My OpenCode**가 **v3.0.0**으로 대규모 업데이트되었습니다! 이번 버전은 에이전트 배포와 관리 방식을 근본적으로 변화시키는 **오케스트레이션 혁명**이라고 할 수 있어요.

## 주요 변경사항

### 동적 에이전트 구성: Categories & Skills

정적인 에이전트 정의에서 벗어나 **동적으로 에이전트를 구성**할 수 있게 되었습니다.

- **Categories**: 모델 추상화 (예: `quick`, `visual-engineering`)
- **Skills**: 특정 작업을 위한 스킬 (예: `frontend-ui-ux`, `git-master`)

💡 **예시:**

```
visual-engineering + frontend-ui-ux = 프론트엔드 전문가
quick + git-master = 빠른 Git 작업 에이전트
```

🎯 **이게 혁신적입니다.** 기존에는 "Frontend UI/UX Engineer" 같은 하드코딩된 에이전트가 있었는데, 이제는 Sisyphus가 동적으로 최적의 에이전트를 조합합니다.

### Prometheus: 전략적 플래너

새로운 **Prometheus** 에이전트가 추가되었습니다.

- 사용자와 "인터뷰"를 진행하며 요구사항을 명확히 파악
- 작업 계획 초안 작성 후 다른 전문 에이전트의 검증을 받음
- 인지 부하를 최소화하면서 계획 정확도를 최대화

⚠️ **Prometheus가 뭔가요?** 그리스 신화의 불을 훔쳐온 신입니다. 여기서는 "전략을 세우는" 역할을 합니다.

### Atlas: 마스터 오케스트레이터

`/start-work` 명령으로 **Atlas** 오케스트레이션 모드를 활성화할 수 있습니다.

| 기능          | 설명                                |
| ------------- | ----------------------------------- |
| 적재적소 배치 | 각 하위 작업에 최적의 에이전트 배치 |
| 끊임없는 검증 | 모든 단계를 작업 계획과 대조 검증   |
| 비용 효율성   | 제공자별로 토큰 사용량 최적화       |

💰 **비용 최적화 예시:**

- 복잡한 계획 수립: Claude Opus (정확도 우선)
- 단순 구현: GPT-4o-mini (비용 절감)
- 코드 검증: Claude Sonnet (균형)

## 제 생각

Oh My OpenCode v3는 **"AI 에이전트의 에이전트"**로 진화했습니다. 단순히 여러 에이전트를 쓰는 게 아니라, **어떤 에이전트를 언제 쓸지 스스로 결정**하는 거죠.

🎯 **이게 왜 중요한가:** 개발자가 에이전트를 일일이 선택하지 않아도 됩니다. Prometheus가 요구사항을 파악하고, Atlas가 최적의 에이전트 조합으로 작업을 분배합니다.

**단점:** 설정이 더 복잡해졌습니다. Categories와 Skills를 이해하고 적절히 구성해야 합니다.

## 업데이트 방법

```bash
# npm
npm install -g oh-my-opencode@latest

# 또는
oh-my-opencode --update
```

⚠️ **주의:** v3는 Breaking Changes가 있습니다. 기존 설정 파일을 백업하고 마이그레이션 가이드를 참고하세요.

Oh My OpenCode v3는 **멀티 에이전트 오케스트레이션의 새로운 표준**을 제시합니다. 아직은 실험적인 기능이 많지만, 방향성은 정말 흥미로워요.

여러분도 Oh My OpenCode를 쓰고 계시다면 v3로 업데이트해보세요. 단, 설정 파일 백업은 필수입니다!
